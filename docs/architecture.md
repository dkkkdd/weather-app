# Архитектура проекта WEATHER-APP

## 1. Общий обзор

Проект построен на модульной архитектуре:
UI → Services → API → Store → Utils.

Цель — обеспечить читаемость, расширяемость и возможность командной разработки.

## 2. Структура директорий

src/
api/ — низкоуровневые сетевые запросы
services/ — бизнес-логика, обработка данных
stores/ — состояние приложения
ui/ — UI-компоненты (JS + CSS)
utils/ — вспомогательные функции
main.js — точка входа
index.html — базовая разметка
style.css — глобальные стили

## 3. Архитектурные принципы

1. UI не вызывает API напрямую.
2. Вся логика работы с API — в services.
3. Store отвечает за состояние и уведомление UI.
4. Компоненты UI независимы и лежат в своих папках.
5. Каждая функция делает одну задачу (Single Responsibility).

## 4. Поток данных

1. UI вызывает service.
2. Service вызывает API.
3. API возвращает данные.
4. Service преобразует их.
5. Store сохраняет состояние.
6. UI подписан на Store и перерисовывается.

## 5. Используемые инструменты

- Vite — сборка
- ESLint — проверка JS
- Prettier — форматирование
- Husky — pre-commit hook
