# Sprint 1 — Weather API App

Цель спринта: реализовать минимальный функционал приложения + настроить рабочий процесс.

## Общие правила спринта
- Каждая задача выполняется в отдельной ветке.
- Название ветки: `feature/<short-name>`.
- Перед созданием ветки всегда делайте `git pull origin main`.
- Любая задача требует Pull Request и Code Review от PM.

---

## Задачи спринта

### 1. Базовая структура проекта (DevA)
- Создать директории:
  - `src/`
  - `src/api/`
  - `src/components/`
  - `src/utils/`
  - `docs/`
- Создать базовый `index.html`.
- Подключить `src/main.js`.

### 2. Функция получения погоды по городу (DevB)
- Создать `src/api/weather.js`.
- Реализовать функцию `getWeather(city)`.
- Обработать статус-коды 200/400/500.
- Вынести API key в `.env-example`.

### 3. Компонент отображения текущей температуры (DevA)
- Создать `src/components/CurrentWeather.js`.
- Отрисовать:
  - температуру
  - иконку погоды
  - название города
- Вызвать компонент в `main.js`.

### 4. Обработка ошибок API (DevB)
- Создать `src/utils/handleError.js`.
- Реализовать человекочитаемые сообщения:
  - “City not found”
  - “API Error”
  - “Network Error”
- Подключить функцию в логику получения погоды.

### 5. Loader при загрузке данных (DevA)
- Создать `src/components/Loader.js`.
- Показывать loader до получения данных.
- Скрывать после получения ответа.

---

## Definition of Done (когда задача считается выполненной)
- Код лежит в отдельной ветке.
- Открыт Pull Request.
- PM оставил комментарии.
- Разработчик исправил замечания.
- PM поставил Approve.
- Ветка смержена в `main`.
